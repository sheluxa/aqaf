# AQA Engine — Прототип генерации тест-кейсов и автотестов

AQA Engine — это прототип ассистента для автоматизации тестирования.  
Он принимает текстовое описание пользовательского сценария и автоматически генерирует:

- структурированный тест-кейс;
- автотест Playwright (TypeScript);
- mock-выгрузку в формате TMS (Qase);
- удобный Web UI для генерации;
- API-доступ через Swagger.

Проект разработан в рамках хакатона как демонстрация концепции «умного помощника тестировщика».

---

# Возможности

### Генерация тест-кейса
Из обычного текстового описания создаётся стандартизированный тест-кейс:

```
Название: …
Предусловие: …
Шаги:
1) …
   ОР: …
```

### Генерация автотеста Playwright
Прототип формирует корректный TypeScript-тест на основе сгенерированного кейса.

### Удобный Web UI
Веб-интерфейс доступен по адресу:

```
http://localhost:8787/
```

Возможности:
- ввод сценария;
- кнопка «Сгенерировать»;
- выбор стиля тест-кейса (краткий / подробный);
- готовые примеры;
- кнопки «Скопировать» для результата.

### Mock‑экспорт в TMS (Qase)
Результат сохраняется в файл `qase_push_mock.json`, показывая, как выглядела бы реальная выгрузка.

### Swagger API
Интерактивная документация доступна по адресу:

```
http://localhost:8787/docs
```

# 1. Установка и запуск

##  Установить Python (3.10+)

Проверь:

```
python --version
```

---

## 2. Установить зависимости

Перейди в папку backend:

```bash
cd backend
```

Создай виртуальное окружение:

```bash
python -m venv .venv
```

Активируй его:

```
.venv\Scripts\Activate.ps1      # PowerShell
```

ИЛИ

```
.venv\Scripts\activate.bat      # CMD
```

Установи зависимости:

```bash
pip install -r requirements.txt
```

---

## 3. Запуск сервера

Команда для запуска:

```bash
python -m uvicorn app:app --reload --port 8787
```

После запуска в терминале появится:

```
Uvicorn running on http://127.0.0.1:8787
```

# Использование

## 1) Web UI

Открой в браузере:

```
http://localhost:8787/
```

Затем:

- введи описание сценария;
- нажми «Сгенерировать»;
- получи тест-кейс и автотест.

---

## 2) Swagger UI

Интерактивная документация:

```
http://localhost:8787/docs
```

Можно тестировать API через браузер.
