# AQA Engine --- Прототип генерации тест-кейсов и автотестов

AQA Engine --- это прототип ассистента для автоматизации тестирования,
который позволяет:

-   генерировать структурированные тест-кейсы из текстового описания,
-   автоматически создавать Playwright автотесты,
-   сохранять результаты (mock-выгрузку),
-   использовать удобный веб-интерфейс,
-   работать через API (Swagger).

Проект создан в рамках хакатона и демонстрирует концепцию «умного»
инструмента для QA/AQA инженеров.

##  Возможности

###  Генерация тест-кейсов

Вводишь текст сценария --- получаешь:

    Название: …
    Предусловие: …
    Шаги:
    1) …
       ОР: …

###  Генерация Playwright автотестов

На основе тест-кейса автоматически формируется TypeScript автотест.

###  Удобный Web UI

Запускается через FastAPI и доступен по адресу:

    http://localhost:8787/

Функционал: - ввод сценария - готовые примеры - выбор стиля тест-кейса
(краткий/подробный) - кнопки «Скопировать» - отображение кода и
тест-кейса

###  Mock-экспорт в TMS (Qase)

Система сохраняет структуру кейса в `qase_push_mock.json`, показывая,
как мог бы выглядеть реальный экспорт.

###  Swagger UI / API

Интерактивная документация и тестирование API:

    http://localhost:8787/docs

##  Структура проекта

    backend/
    │
    ├── app.py                 # Веб-сервер и UI
    ├── requirements.txt       # Пакеты Python
    │
    ├── services/              # Логика генерации и mock-экспорт
    │     ├── generation.py
    │     ├── qase_client.py
    │
    ├── models/                # Pydantic-схемы
    │     └── schemas.py
    │

##  Установка и запуск

### 1) Клонируй репозиторий

``` bash
git clone <url>
cd hackathon-aqa-mvp/backend
```

### 2) Создай виртуальное окружение

``` bash
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 3) Установи зависимости

``` bash
pip install -r requirements.txt
```

### 4) Запусти сервер

``` bash
python -m uvicorn app:app --reload --port 8787
```

### 5) Открой в браузере:

    http://localhost:8787/

##  Env-файл

Создай в папке `backend`:

`backend/.env.example`

    QASE_API_TOKEN=
    QASE_PROJECT_CODE=
    QASE_SUITE_ID=
